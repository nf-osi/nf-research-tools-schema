Context

The Challenge: Fragmented Research Infrastructure

NF research spans dozens of institutions, disease manifestations, and model systems. Without a common registry, tools developed with public funding are:

  • Rediscovered or rebuilt independently at each institution
  • Described inconsistently across publications
  • Difficult for new investigators to locate and request
  • Hard for funders to track as research outputs
  • Invisible to the patient and clinical communities

The goal: A single, structured, publicly accessible registry of every research tool produced by the NF community — so tools can be found, cited, and reused.

These submission forms are the front door: the standardized interface through which researchers contribute entries to that registry, hosted on Synapse.

Tool Categories · Infrastructure

Biobanks

Biobanks are repositories that store and distribute NF biospecimens — tumor tissue, blood, DNA, RNA, and cell preparations. Rather than a single experimental tool, a biobank entry makes an entire collection of specimens discoverable.

🏦

Biobank Form Fields

Required (beyond universal)
Biobank URL
Notable Optional Fields
NF disease types covered (NF1, NF2, Schwannomatosis) Specimen types (tissue, blood, DNA, RNA, cells) Tumor types available Preservation methods (FFPE, cryopreserved, flash-frozen) Specimen formats Access procedure & request form URL Biobank website URL Curator / contact name & affiliation MTA required NGNRI repository affiliation

4 NF biobanks are already registered, making specimen access discoverable through a single interface for the first time.

Biobank entries require a name and URL — enough for the community to find and contact the biobank immediately. All specimen and content details can be added over time.

Part 2 · Observations

Linking Experimental Findings to Tools

Beyond cataloging tools, the registry captures how they perform — connecting specific experimental findings from publications to the tools that generated them. This transforms the registry from a directory into a living evidence base.

1️⃣

Select Tool & Observation Type

Dropdowns pre-populated with all existing registry entries by category. Observation types are context-sensitive: a cell line offers drug response, viability, gene expression; an animal model offers tumor growth, survival, behavioral, drug efficacy.

2️⃣

Describe the Finding

Free-text description of the specific experimental result or observation, plus the DOI of the paper it comes from.

3️⃣

Tag with Ontology (Optional)

Phenotypic observations can be tagged with Mammalian Phenotype (MP) ontology IDs, enabling cross-study comparisons.

4️⃣

Attach Files (Optional)

Supporting images, datasets, or figures can be uploaded alongside the observation record.

Strategic Value

Why This Matters for the Funding Organization

📊

Portfolio Visibility

Every grant-funded tool is structured, tagged, and searchable — giving program officers a real-time view of what the portfolio has produced, filterable by disease area, model type, or institution.

♻️

Maximizing Return on Investment

When tools are registered with acquisition instructions, other funded investigators can access and reuse them immediately — reducing duplication and stretching grant dollars further.

📎

Publication Tracking

The registry links each tool to every paper that has used it. Program managers can pull a citation map showing downstream research productivity enabled by any specific funded tool.

🤝

Community Coordination

Biobanks, clinical assessment tools, and patient-derived models are visible across institutions — enabling the program office to actively broker resource sharing and cross-institutional collaborations.

✅

Grant Reporting Support

Grantees can point to registry entries as structured evidence of deliverables met. Program officers can generate standardized tool-output reports directly from the registry.

The submission forms are more than data entry. They are the mechanism by which the organization's investment in tools becomes a permanent, shared, and growing community asset.

What Every Submission Captures

  • Tool identity & scientific description
  • NF disease relevance & model system
  • Publication that first described it
  • All publications that have used it
  • How another lab can access it
  • RRID for durable citation tracking
  • Experimental observations from literature
